# RaceFrame Admin UI Style Rules

This note captures the styling direction currently implemented for:

- `/admin`
- `/admin/events/new`


Use this as the default UI rule set for future admin-side work unless the user asks to change direction.

## Overall direction

- Keep the UI simple and restrained.
- Prefer a clean editorial layout over heavy dashboard chrome.
- Avoid "AI-ish" generic card stacks and decorative filler copy.
- Use spacing, lines, and typography more than boxes.
- Keep pages visually related to the RaceCraft Labs reference style.

## Header rules

- Header background should use the dark RaceCraft-style teal/navy tone already defined in CSS as `--header-bg`.
- Header should be simple:
  - `RaceFrame` on the left
  - navigation links on the right
- For admin-related pages, header nav currently uses:
  - `Upload`
  - `User`
- Do not reintroduce the ribbon/marquee under the header for admin/create/edit pages.

## Background rules

- Use the light warm background already implemented in `admin.css`.
- Keep the diagonal/slanted background lines very subtle.
- Do not switch admin pages back to dark full-page panels.

## Typography rules

- Use the existing font system from `admin.css`:
  - `Barlow Condensed` for headings and buttons
  - `Lora` for body text
  - `Fira Code` for labels, small metadata, and utility text
- Headings should be bold and compact, but not oversized.
- Metadata such as slugs should be smaller and lighter than primary content.

## Layout rules

- Prefer one clear page structure.
- Keep generous vertical spacing between:
  - page header
  - stat/action strip
  - main content table or form
- Avoid unnecessary explanatory sections, captions, or extra CTA blocks.
- For simple pages, do not add side panels unless the page function truly needs them.

## Dashboard rules

For `/admin`:

- Hero area should stay minimal.
- Main action is `Create Event`.
- Stats should live in one thin horizontal strip.
- Current events should be shown in a clean table/list.
- Event slug should sit under the event name in smaller lighter text.
- Status should be shown as light pill badges:
  - published: green
  - draft: red/rose
- `Open` should look like a boxed action, not plain text.
- Do not place `Back to Dashboard` on the dashboard itself.

## Create page rules

For `/admin/events/new`:

- Keep the same header style as the dashboard.
- Include a `Back to Dashboard` button on the create page.
- Keep the form visually simple.
- Do not show the autogenerated slug caption block.
- Avoid extra helper copy unless it is necessary for form completion.

## Button rules

- All buttons must have a visible hover state.
- Hover states should change background/border clearly enough to notice.
- Primary buttons use the gold-accent treatment.
- Secondary buttons use the soft teal treatment.
- Danger buttons use the rose/red treatment.

## Table rules

- Tables should be clean and readable.
- Use subtle separators, not heavy boxes.
- Headers should use small mono uppercase styling.
- Keep action cells compact and obvious.

## Avoid

- Large rounded container stacks everywhere
- Dark full-page admin backgrounds
- Excessive helper paragraphs
- Duplicate navigation actions
- Decorative ribbons on admin/create/edit pages
- Generic dashboard filler sections

## Main implementation files

- `backend/app/templates/base.html`
- `backend/app/templates/admin_dashboard.html`
- `backend/app/templates/admin_event_form.html`
- `backend/app/static/admin.css`
